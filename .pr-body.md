Establish the DeFi Vault Intelligence Platform monorepo scaffold per the [Coding Guidelines](https://www.notion.so/Coding-Guidelines-Working-with-Claude-Code-316d81f6e2628129a496f8df02aa77da): backend (FastAPI + SQLAlchemy + Alembic), workers (Celery), frontend (Next.js 14 App Router), infra (Docker Compose + Nginx), and docs placeholders.

**Why:** Sprint 1 requires this layout before implementing auth, API routes, and protocol adapters. Single initial commit keeps the scaffold reviewable.

**Scope:**
- Root: `.gitignore`, `.env.example`, `README.md`
- Backend: app factory, config, async DB, API stubs (auth, wallets, positions, vaults, admin, ws), services (pnl, risk, advisory, prices), protocol adapters (abstract base + morpho/aave/pendle/euler/registry), Alembic, pytest fixtures
- Workers: Celery app, config, task stubs (prices, vault_metrics, positions, risk, advisory, reconstruction, cost_monitoring)
- Frontend: Next.js 14 + TypeScript + Tailwind, `src/app`, `src/lib/api.ts`, placeholder dirs
- Infra: `docker-compose.yml` (PostgreSQL/TimescaleDB, Redis, backend, Celery worker/beat, frontend, Nginx), nginx config, backup/seed scripts
- Docs: `deployment.md`, `troubleshooting.md`, `monitoring.md` placeholders

No behavior beyond “hello world” surfaces; all services are stubbed for follow-up tickets.
